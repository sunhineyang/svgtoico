# 实现转换历史页面和知识页面

## 任务概述
为 SVG 转 ICO 工具添加两个新页面：
1. **转换历史页面** - 使用 localStorage 存储和展示用户转换记录
2. **知识页面** - PPT 风格的图文并茂内容，介绍 SVG/ICO 等知识

## 实现步骤

### 第一部分：转换历史页面

1. **创建数据存储层**
   - 创建 `src/hooks/use-conversion-history.ts` - 管理历史记录的 Hook
   - 创建 `src/lib/history-utils.ts` - localStorage 工具函数
   - 创建 `src/store/history-store.ts` - Zustand store（可选，也可用 Hook）

2. **创建历史页面组件**
   - 创建 `src/app/[locale]/history/page.tsx` - 路由页面
   - 创建 `src/components/pages/history-content.tsx` - 主组件
   - 创建 `src/components/history/history-card.tsx` - 历史记录卡片
   - 创建 `src/components/history/history-stats.tsx` - 统计信息
   - 创建 `src/components/history/history-empty.tsx` - 空状态

3. **集成到转换流程**
   - 修改 `src/store/converter-store.ts` - 在转换成功时保存到历史
   - 修改 `src/components/converter/conversion-result.tsx` - 添加"保存到历史"功能

4. **添加导航链接**
   - 修改 `src/components/layout/header.tsx` - 添加 History 导航链接
   - 修改 `src/components/layout/footer.tsx` - 添加 History 链接

5. **添加多语言支持**
   - 更新 `messages/en.json` - 添加 history 相关翻译
   - 更新 `messages/de.json`、`messages/es.json`、`messages/ja.json`、`messages/ko.json`、`messages/ru.json`

### 第二部分：知识页面（PPT 风格）

1. **创建知识页面组件**
   - 创建 `src/app/[locale]/guide/page.tsx` - 路由页面
   - 创建 `src/components/pages/guide-content.tsx` - 主组件
   - 创建 `src/components/guide/slide.tsx` - 单张幻灯片组件
   - 创建 `src/components/guide/slide-navigation.tsx` - 导航按钮
   - 创建 `src/components/guide/slide-progress.tsx` - 进度指示器
   - 创建 `src/components/guide/slide-sidebar.tsx` - 幻灯片目录

2. **实现幻灯片内容**
   - 设计 10-12 张幻灯片的内容结构
   - 每张幻灯片包含：标题、内容、图标/插图
   - 使用现有的 UI 组件（Button、Card 等）

3. **实现交互功能**
   - 左右箭头切换
   - 键盘快捷键支持（← →）
   - 底部进度指示器
   - 自动播放模式
   - 全屏模式
   - 幻灯片目录导航
   - 记忆浏览位置（localStorage）

4. **添加导航链接**
   - 修改 `src/components/layout/header.tsx` - 添加 Guide 导航链接

5. **添加多语言支持**
   - 更新所有 `messages/*.json` - 添加 guide 相关翻译

### 第三部分：样式和动画

1. **历史页面样式**
   - 卡片式布局
   - 悬停效果
   - 响应式设计

2. **知识页面样式**
   - 每张幻灯片独立配色
   - 渐变背景
   - 平滑过渡动画（淡入、滑动）
   - 响应式适配移动端

## 技术要求

- 使用 TypeScript
- 遵循现有代码风格
- 使用 Tailwind CSS
- 使用 next-intl 实现多语言
- 使用 Zustand 或 React Hooks 管理状态
- 使用 localStorage 存储历史记录（注意容量限制，可考虑压缩或只存储元数据）
- 响应式设计，支持移动端

## 注意事项

- localStorage 容量限制约 5-10MB，历史记录的 ICO 数据可能较大，考虑：
  - 只存储最近 N 条记录（如 20 条）
  - 不存储 ICO 数据，只存储元数据，重新下载时需要重新转换
  - 使用压缩算法减少存储空间
- 幻灯片内容要图文并茂，使用 Lucide React 图标
- 保持与现有页面风格一致
- 确保所有新页面都支持多语言
